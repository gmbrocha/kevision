ScopeLedger walkthrough guide - updated April 30, 2026

Web App Walkthrough

Turn drawing revisions into accountable scope records.

This guide is written to sit beside the live app during a demo or working review. It explains what each page is for, what the reviewer is deciding, and what ScopeLedger is doing behind the scenes.

Demo URL: http://127.0.0.1:5000
Landing page: Projects. Choose the active project before opening project-scoped pages.
Current active project: Demo Project
Primary story: select a project, import or append revision packages, review detected changes, verify drawing context, and export a pricing-ready workbook.

Fast Demo Path

1. Projects Select Demo Project, create future projects, stage initial package files, or restore archived work.
2. Overview Show the selected project's progress, stage package files, run Populate Workspace, and check export readiness.
3. Review Changes Open the review queue. Filter, search, bulk accept/reject, or review one record at a time.
4. Change Detail Use the cockpit to inspect source evidence, edit workbook text, and accept or reject the record.
5. Drawings Show active and superseded drawing versions and open a sheet for full context.
6. Sheet Detail Inspect a high-resolution drawing page with detected regions and a version chain.
7. Latest Set Verify the conformed latest drawing set and confirm index-like pages are not driving the answer.
8. Export Workbook Generate and open the workbook, review packet, conformed preview, or Drive folder.
9. Diagnostics Check PDF health and ingestion warnings without mixing technical warnings into scope decisions.

Core Concepts

ScopeLedger keeps a local workspace for each project. The workspace stores imported packages, sheet versions, detected cloud regions, generated change items, reviewer decisions, exports, and diagnostic data.

The product value is not just detection. The value is turning noisy revision evidence into accountable records that can be reviewed, edited, accepted, rejected, and exported.

Data object Meaning in the app
Project A top-level job or contract. Counts, packages, drawings, review queue, and exports are scoped to the selected project.
Revision package A folder or PDF set imported into a project. Additional PDF files can be appended from the Overview page.
Sheet version A single drawing page from a package. ScopeLedger tracks whether it is active or superseded.
Detected region A CloudHammer visual candidate or source region that may represent changed scope.
Change item The reviewable record that becomes workbook text when accepted.
Export artifact The workbook, review packet, conformed preview, CSV/JSON support files, and diagnostics.

Projects

Projects is the top-level workflow boundary. Use it to choose which job the rest of the app is showing. Once a project is selected, Overview, Review Changes, Drawings, Latest Set, Export, and Diagnostics all load that project's workspace data.

What to point out

  • Selected project: the active project driving every downstream page.
  • Create project: adds a new project with optional initial package files from the user's browser.
  • Workspace folder: optional server-side destination for generated workspace data.
  • Archive: removes completed work from active workflow tracking without deleting the project workspace.
  • Project archive: shows archived projects and lets them be restored.
  • Restore: brings an archived project back into the active list.
Demo phrasing: "This is now structured as a project system, not a one-off demo. Demo Project is the current workspace, but the same workflow can be repeated per job."

Overview

Overview is the selected project's dashboard. It shows review progress, the active package, project packages and files, package import controls, Populate Workspace, and export readiness.

What to point out

  • Review next: jumps directly to the next pending record when work remains.
  • Pending review: records still awaiting a human decision.
  • Accepted / Rejected: decisions already made by the reviewer.
  • Needs check: pending items with low signal, placeholder text, or other concerns.
  • Import package: opens a browser file dialog for the user's computer to select PDFs or a folder, then stages them into the selected project.
  • Manual server path: remains available for server-local paths when the person using the browser is also on the host machine.
  • Append PDF: opens a browser file dialog for the user's computer to add PDF files to an existing package.
  • Populate Workspace: runs the backend scan over staged source files to generate sheet versions, drawing renders, diagnostics, and change items.

This page is also the right place to explain that all stats are project-specific. Switching projects changes the package list, drawing set, review queue, and exports.

First-run workflow: create/select project, stage one or more package PDFs, click Populate Workspace, then review the generated queue. Population is the long-running backend step.

Review Changes

Review Changes is the queue. It is designed for fast triage: filter records, search by sheet or text, bulk accept or reject obvious rows, or open a single item for detailed inspection.

Control Use
Search Find by sheet number, detail reference, detected text, reviewer text, or provenance.
All / Pending / Accepted / Rejected Switches the queue status view. Accepted and rejected rows are still visible for audit.
Needs check Shows pending records that deserve manual attention before final export.
Accept selected / Reject selected Applies a bulk decision to checked rows.
Review Opens the Change Detail cockpit for source-backed review.
Status language: Accepted means the record is approved for workbook use. Needs check is an attention flag on pending work, not a second final status.

Change Detail Cockpit

This is the main review screen. The left side shows image evidence. The right side shows sheet metadata, revision context, editable workbook text, reviewer notes, and decision buttons.

Reviewer workflow

  1. Inspect the crop or sheet image on the left.
  2. Use mouse wheel to zoom inside the image pane. Drag to pan the zoomed image. Double-click to reset.
  3. Confirm the sheet number, cloud/detail reference, package, discipline, and sheet title.
  4. Edit Scope of change into the text that should appear in the workbook.
  5. Add reviewer notes when the item needs context, follow-up, or field verification.
  6. Click Accept + Next for valid scope or Reject + Next for noise.

Keyboard shortcuts

A Accept the current item and advance.
R Reject the current item and advance.
S Save text and notes without changing queue position.
[ / ] Move to the previous or next queued item.
Image quality note: viewer panes now use cached high-resolution PDF renders. Full sheets and cloud crops are regenerated for viewing so zooming is much more legible than the original compressed page/crop assets.

Drawings

Drawings is the sheet index for the selected project. It shows discovered sheet versions, status, discipline, package context, change count, and a direct link to each sheet's full drawing page.

Column Meaning
Sheet The drawing number. Long sheet-index text is compacted so the list remains readable.
Revision / Previous The visible revision context available for that sheet version.
Discipline Inferred from drawing number prefix for quick review context.
Changes Number of generated change items tied to that sheet version.
Status Active sheets represent the latest known version; superseded sheets are preserved for audit.

Use this page to demonstrate that ScopeLedger tracks sheet versions, not just a flat list of extracted notes.

Sheet Detail

Sheet Detail is the full drawing context page. It shows a high-resolution rendered drawing page, overlays detected regions, and keeps a right-side panel for version chain and changes on that sheet.

Use this page when

  • A crop is too narrow and surrounding sheet context matters.
  • You want to show how workbook rows connect back to drawing evidence.
  • You need to verify whether a sheet is active or superseded.
  • You need to move between detected regions and Change Detail review.

The image pane supports wheel zoom, drag pan, and double-click reset. Clicking a highlighted region opens the related Change Detail page.

Latest Set

Latest Set is the conformed drawing view. For each drawing number, ScopeLedger shows the latest version it detected and the prior versions it supersedes.

Important details

  • Revised only: focuses the view on sheets with more than one version.
  • All sheets: shows the full detected latest set.
  • Revised flag: marks sheets that changed across revision packages.
  • Revision chain: shows latest revision and immediate prior revisions.
  • Index-like pages: sheet-index matches are deprioritized so pages like AE102 do not incorrectly open an index page when a real sheet version exists.

This page supports the pricing handoff story: before sending a workbook, the team can confirm the system is using the correct latest sheet versions.

Export Workbook

Export is the handoff page. It centralizes generated artifact links so the demo can move cleanly from review to deliverables.

Direct artifact buttons

Button What opens
Open Workbook The generated Excel workbook with a summary cover sheet and detailed changelog.
Open Review Packet An HTML review packet with change crops and source context.
Open Conformed Preview A PDF preview of the latest detected sheet set.
Open Google Drive Folder The configured review folder for sharing or file handoff.

The export history table records prior export runs, accepted/pending counts, and whether the export was standard or interim.

Diagnostics

Diagnostics is deliberately separated from the review workflow. It is for PDF health, ingestion warnings, and troubleshooting. The goal is to catch malformed PDFs or text-layer issues without confusing those issues with scope decisions.

Area What it tells you
PDF health Each source PDF, page count, warning count, issue count, and max severity.
Issue summary Grouped counts by severity and issue type, instead of a huge noisy list.
Filtered technical details Available when troubleshooting is needed, but not the primary review surface.

The key demo message is that diagnostics are available, but the normal reviewer does not need to adjudicate every low-level PDF warning.

Status Meanings

Status or flag Meaning Export behavior
Pending The item has not been adjudicated yet. Visible as unresolved review work. It may require an interim export decision.
Accepted The reviewer believes the item is valid scope. Included as approved scope in workbook/export records.
Rejected The reviewer intentionally excluded it. Preserved in workspace data for audit, but not treated as approved scope.
Needs check An attention flag on a pending item with low signal, placeholder text, or another concern. Should be manually reviewed before final handoff.

Current Limits

ScopeLedger is already useful as a structured review and export surface, but the demo should be precise about what is and is not fully wired.

  • Cloud/crop evidence is wired into the web app and now uses high-resolution viewer assets.
  • Accepted/rejected review state is persisted in the local project workspace.
  • Sheet versioning, active/superseded state, project selection, archive/restore, package staging, and Populate Workspace are wired.
  • OCR and final scope-text extraction are still in progress. Some detected text may be placeholder or source-adjacent text.
  • Diagnostics explain PDF ingestion health; they are not a punch list of field fixes.

Security And Local Review Note

The app runs locally against the project workspace. Review decisions, source paths, generated crops, cached high-resolution images, and exports are stored in local workspace data. The review assist endpoint is disabled while model integration is in progress, so the web app is not making new API verification calls from the review screen.

For a meeting, the clean security message is: ScopeLedger can produce a structured workbook and review packet from project drawings while keeping review state and source context auditable in the local workspace.
