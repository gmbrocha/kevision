# ScopeLedger

Turn drawing revisions into accountable scope records.

Current focus: train CloudHammer into a reliable cloud detector/cropper. Review
work should happen only when it creates training/eval artifacts or validates the
cropper after the detector.

Start here:

- `ROADMAP.md` - active sequencing and guardrails
- `CLOUDHAMMER.md` - current model/training source of truth
- `SCOPELEDGER.md` - repo structure, setup, active workflow, architecture boundaries
- `PRODUCT_AND_DELIVERABLE.md` - Kevin-facing product/workbook rules and future scope/detail work
- `docs/SECURITY_PRIVACY_POLICY.md` - ESA-facing security/privacy policy

Older handoffs, epoch docs, and deprecated compatibility code live under
`archive/`. Do not treat archived files as current guidance unless one of the
maintained docs explicitly points there for historical context.

Dated meeting notes under `docs/meetings/` and question/history notes under
`docs/history/` are reference records, not active sequencing. Current direction
lives in the maintained docs listed above.
